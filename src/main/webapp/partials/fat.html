About Application

  • Based on simple Maven Java web app with AngularJS for front end(Reload content of a page if data changes in background).

  • On server side it uses WebSocket to push stock updates when there is a change to all clients connected to this Server/WebSocket between 09:15 to 15:15 everyday assuming that these are the working hours of Stock Market.

  • To generate a new prices for Stocks there is a Producer thread which keeps on running in background and gets started when ServletContext gets initialized. I have written a class that implements ServletContextListener from where I'm starting my Producer Thread.

  • This thread pushes JSON data every 10 Seconds in string format over the socket after generating new set of Stock prices in each iteration using WebSocket reference which is declared as a member variable.

  • Wrapper format of data sent is as follows:
    {"chartColumns":[{"id":"month","label":"Month","type":"string"},
    {"id":"bmw","label":"BMW","type":"number"},
    {"id":"massarati","label":"Massarati","type":"number"},
    {"id":"pagani","label":"Pagani","type":"number"}],
    "message":"MARKET_NOT_STARTED/DATA_AVAILABLE",
    "chartData":"array of json",
    "status":"NO_DATA/SUCCESS"}

  • On client side, AngularJS needs data in specific JSON format to render a chart based on that data. So Producer thread generates.

  • It uses library to render these charts Connection to web socket in done in directive which is responsible for drawing chart once data gets updated using onMessage method.
